E3A OMS

An all-in-one internal platform to streamline order processes and enhance team collaboration

#Web App

Project Overview

This project focuses on improving the user interface of the company’s internal order management platform, used by the marketing team and its members. The original system had a complete interface, but it was complex and hard to navigate, leading to frequent user issues and difficulty finding information.

The Problem

At the start of the project, we collaborated with the marketing team to review their requirements. While the existing system included all necessary features, usability issues impacted efficiency. These included a lack of visual hierarchy, causing important information to be overlooked, and cumbersome workflows for frequent tasks.

Project Goal

  • Improve clarity of the overall information structure
  • Enhance visual hierarchy and reading rhythm across key pages
  • Streamline workflows to reduce steps and improve efficiency

My Role

  • Analyzed detailed requirements provided by the marketing team and identified key design priorities
  • Redesigned the system’s information architecture and optimized UI layout and interaction logic
  • Created high-fidelity UI mockups to ensure visual clarity and usability

1. Optimizing Requirements

Discovering Opportunities for Improvement

During the project initiation, the marketing team created a detailed requirements doc, highlighting issues with the existing order management system and features to be added or optimized. The list covered several modules, including account permissions, device management, order management, inventory, and coupon management.

Based on the requirements list, I worked with the development team to understand the scenarios and assess feasibility. We then optimized the chaotic and inconsistent interface. To align the design with actual needs, I categorized and refined the feature requirements into clearer, actionable design directions. Below are key details from the requirements list, focusing on the Order and Device Management modules.

Order Management Module

Requirement Details Priority
1 Order overview, displaying basic order information in a list, including order number, customer phone number, location, rental time, selected package, order status, and payment status. 0
2 Ability to perform fuzzy searches using keywords like order number, customer phone number, device number, and time range, with multiple filtering options. 0
3 Ability to calculate the total amount based on the package billing rules and actual usage time. If there are exceptions or special cases, administrators can manually adjust the amount and record the reason for the change. 1

Device Management Module

Requirement Details Priority
1 Support for single or bulk device entries. Devices can be added through the system interface, including custom unique device codes, specifications, and manufacturing dates. 1
2 Administrators can bind rental packages or products to devices individually or in bulk, as well as set and push ads to the devices. 0

2. Order Management Module Optimization

Optimizing the Order Management Experience

Through user interviews and system observations, I identified issues with scattered information, confusing hierarchy, and inconsistent styles in the original order management and device list interfaces. I reorganized the structure and interaction logic, focusing on optimizing the display of key information and improving operational efficiency. Below is a comparison of the interface before and after optimization.

Order Interface

As shown in the image (click to enlarge), the current interface lacks key features, including: ① an order search function, ② proper display of order details, requiring extra time to check overall information, and ③ a clear visual representation of order status, making it hard for users to quickly assess the order's progress.

Optimization ① - Sidebar Indentation Feature

Indented the sidebar to improve readability and optimize space.

Optimization ② - Order Details Modal

Order details are now displayed in a pop-up modal, offering a focused view of relevant information and functions.

Optimization ③ - Feature Additions and Color Adjustments

① Updated with the E3A theme color and company logo; ② Added order search and filter functions; ③ Expanded order status options to cover more scenarios; ④ Introduced additional features for orders.

Order Settlement Process

Users can settle devices by viewing the total amount based on package rules and usage time. In special cases, they can manually adjust the amount and add notes to record the reasons for changes.

Adjusting the Amount for Special Cases

In case of issues like network instability, errors may occur when adjusting the amount, and the confirmation button will become disabled until the issue is resolved.

Confirming the Settlement

If the amount is adjusted successfully but an issue occurs during confirmation, an error modal will appear. After clicking cancel or waiting 800ms, the user returns to the main interface, with the order’s settlement status marked as "Settlement Error."

3. Device Management Module Optimization

Enhancing the Device Management Experience

The device module optimization focuses on adding and binding devices. The new design offers more intuitive visual information in the device list and uses modal pop-ups for adding and binding devices and packages, enabling users to manage updates more efficiently.

Adding a Single Device (also applicable for multiple devices)

The new "Add Device" modal, shown in the image, supports adding multiple devices at once, with fields for multi-device entries. Users can select device types, define names and images, and set location coordinates. The second page allows users to add agent information and device-specific settings.

Device Package Binding Process

This feature allows users to remotely bind and modify device packages. When selecting a device, a corresponding package menu appears (currently for jaundice meter devices), with pagination and search functionality in the modal.

Reflection

The E3A OMS Optimization project aimed to enhance user experience and operational efficiency. As a B2B project, the focus was on functional optimization and process simplification rather than visual design. I optimized the existing interface based on the marketing team's requirements and designed new UI features and workflows. Through collaboration, we addressed issues like complex interfaces and cumbersome operations. The optimized system has received positive feedback, with smoother operations, improved efficiency, and higher user satisfaction. This project has sharpened my design skills and provided valuable experience in cross-team collaboration and quickly responding to needs.
