Voop

A professional software for auto-generating business proposals

#Software

Project Overview

Voop is a tool that helps companies automate proposal creation. It offers marketing teams and project managers a more efficient way to collaborate and organize proposals — saving time by removing the need to search for old content, rewrite similar sections, and adjust formatting manually.

Goal

Simplifies the proposal creation process with template selection, auto-filled content, and customizable design. It also supports role-based features and permissions tailored to different user types, such as marketers and business managers.

My Role

  • Collaborated closely with the development team to provide design solutions and help define core product features
  • Created MVP based on project requirements, outlining layout and user flows
  • Designed interactive prototypes in Figma to showcase key functionalities
  • Collected and organized user feedback to iterate and refine the design

1.寻找问题

Finding the problem

John S.

年龄：35

职位：项目经理

部门：项目管理部门

技能：项目规划、项目协作、客户沟通、时间管理

目标

确保项目按时按预算完成，同时与客户和团队保持良好沟通。

需求

  • ① 频繁编写和更新项目提案，尤其是在时间线、预算和风险等方面，导致大量的重复工作。
  • ② 与市场团队协作时，常常面临提案标准不一致的问题，导致修改提案时需要额外的时间和精力。
  • ③ 提案格式需要根据不同客户和项目需求进行调整，增加了大量的手动修改工作。
  • ④ 客户需求频繁变化，导致团队难以迅速应对调整，影响项目进度。
  • ⑤ 团队成员，尤其是新人，难以快速适应现有流程，需要一个易于上手的工具。

情景

John 经常面临因重复性工作而浪费大量时间，尤其是在与市场团队合作时，提案的标准和格式不一致，导致沟通效率低下。每次需要重新修改提案时，他都感到十分沮丧，因为每次修改都会影响团队的进度。

“我们需要更高效的工具减少重复性工作，确保团队协作顺畅，同时能够快速适应客户需求的变化。”

Emma J.

年龄：32

职位：市场营销经理

部门：市场部门

技能：市场调研、品牌建设、客户分析、提案撰写

目标

创建高质量、定制化的市场营销提案，吸引新客户并促成交易。

需求

  • ①每次编写营销提案时，需要大量寻找、修改和格式调整，工作繁琐且重复。
  • ②提案内容需要根据客户需求灵活调整，现有工具不能高效支持这些变化。
  • ③提案编写耗时，导致没有足够的时间进行市场调研和品牌策略制定。
  • ④团队成员之间缺乏协作工具，导致文档反复修改，沟通效率低下。

情景

Emma 经常需要花费大量时间修改提案格式和内容，特别是在应对客户不断变化的需求时，常常感到时间不够用。由于现有工具的不灵活，她很难在提案的个性化和高质量之间找到平衡。

“我需要一个能够自动化生成提案内容，并且快速适应客户需求变化、且能整合已有提案素材的工具，这样我能专注于战略性工作。”

痛点

Pain Point

"我通过与多个市场营销团队和项目经理的访谈，了解了他们在编写提案文档时的痛点，主要包括提案内容的重复性工作、格式调整、以及根据客户需求进行个性化修改的挑战。这些问题促使我设计了Voop项目，旨在通过模板化和自动化生成提案文档来提升效率和质量。"

痛点 #1: 冗余

Voop这个软件的项目创立提议来自于我们的高层会议，根据我们PM的描述，写提案的团队对提案的内容创作其实大部分时间是在整理格式以及拼接一些过往资料。大量的精力会浪费在改客户名、调格式、整理结构上而非内容的本身 。

痛点 #2: 混乱

来自不同团队的文档风格差异巨大，有的直接用Word，有的PPT中图文混乱，有的甚至直接发PDF截图。 不统一的文件格式堆积出来的后果就是让整个公司在客户面前缺乏一致性和专业形象，而这会直接影响到潜在客户对公司的判断。

痛点 #3: 迷惑

虽然公司内部有一套看似完整的提案模板和撰写指南，但根据市场团队的成员反馈，面对这些没有完整结构化的“模板”时，有人告诉他们内容应该从哪里写起、信息该怎么组织，于是模板变成了“摆设”，每份提案仍旧各自为政，既拖慢流程，也削弱了品牌专业感。

2.构思

It's brainstorming time

起点

根据上面的痛点，我能很好的了解到使用者们不是不会写，而是不知道从哪开始、怎么构建结构。所以我一开始的目标，并不是设计一个炫酷的模板，而是构建一个流程感强、逻辑清晰的提案生成方式。

User Flow

这张使用流程图展示了用户在 Voop 系统中建立或编辑提案的完整路径。从登录开始，用户可以选择新建或修改提案，并通过模板与数据库辅助完成内容撰写。流程中也设计了审核与导出机制，确保提案的效率与质量，同时贴合企业真实工作流程。

Sitemap

这张网站地图（Sitemap）规划了 Voop 系统的整体信息架构。从登录流程开始，根据不同用户角色（如提案方、审核方、后台管理员）设计了对应的功能模块与页面层级，包含提案建立、内容管理、审核流程与数据分析等模块。通过清晰的结构安排，提升用户的导航效率，也为后续的界面设计与开发奠定了基础。

3.构建

Wireframe

这一部分主要展示了 Voop 的基础使用流程，聚焦在 MVP 阶段的核心功能。从登录开始，用户依次进入主页、创建提案、选择文案模板、进入编辑区，最终导出提案。 每一步都贴近用户的真实需求，具备清晰、合理的操作流程体验，为后续的功能扩展打下基础。

① 用户通过主界面进行登陆； ② 登录后会来到Dashboard区域，然后选择新建proposal。 *修改已有Proposal的方式则为在上方区域点击需要修改的目标文件即可。

① 选择Proposal中需要用到的问题，逻辑是通过选取需要的问题自动生成整体大纲后进行细节修改（由公司数据库提供）；② 输入Proposal的基本信息；③ 下一步； ④ 选择喜欢的template。

*这里分成了两个不同的情况，绿色线路为高级别的用户，橙色则为普通等级用户。 绿① 假设修改完成后按左侧导出键进行导出； 绿② 选择符合使用者需求的选项后进行导出； 绿③ 完成导出。 橙① 低级别用户按左侧导出键弹出提交申请权限的Modal； 橙② 权限获批后开放导出按钮（等待期间用户可以通过左侧工具栏查看申请状态/导出按钮查看进度）； 橙③ 剩余流程与绿色线路一致。

回顾

Voop 是一款面向市场团队与项目经理的提案文档生成工具，旨在解决格式不统一、内容更新繁琐、协作流程不清晰等问题。在设计过程中，我们以用户体验为核心，结合访谈与流程梳理，明确了模板灵活性、内容自动化、权限控制与协作等核心需求。通过多轮线框图与原型测试，我们优化了功能结构与操作流程，提升了团队协作、进度展示、数据导入与模板选择等体验。这个项目让我更加理解如何从实际场景出发进行设计，并在功能与体验之间取得平衡，同时也提升了我在组件化设计、与开发协作以及复杂界面统一性上的能力。
