E3A IntelliCare Platform

An Integrated App Connecting All E3A Healthcare Smart Devices for Women and Newborns

#Mobile App

Project Overview

This project introduces a unified platform for E3A Healthcare, allowing users to connect and manage all smart devices within a single app.

Problem

Users currently need separate apps to operate different E3A Healthcare devices. The inconsistent interfaces and interaction patterns make device management more complicated.

Goal

Create a unified smart device management platform where users can connect and control all E3A Healthcare products within a single app, improving convenience, device collaboration, and the overall brand experience.

My Role

  • Led the UI design of the project and built the visual system from the ground up
  • Designed a unified interface style to ensure a consistent experience across different products
  • Defined interaction guidelines to improve usability and workflow efficiency
  • Participated in requirement analysis and refined solutions based on user needs
  • Collaborated with product managers and developers to implement and iterate design

1.Discovery

From Business Goals to User Needs

As E3A Healthcare continues to expand its smart devices for women and newborns, the core products — a jaundice meter and a fetal heart monitor — still rely on separate apps. The differences in interface design and interaction patterns add extra effort for users and create a fragmented brand experience.

To solve this issue and support future devices, the team built a unified platform for women and newborns, offering a consistent and seamless experience. Its goals are to strengthen brand recognition through a unified design and to centralize devices in one app, reducing user effort and improving efficiency.

4.Deliver

Key Interface Designs

Based on the wireframes and the product direction discussed with the team, I defined the overall visual style and built core components to ensure consistency and reusability. On this foundation, I developed the high-fidelity UI, with selected key screens shown below.

Key Interfaces & Feature Highlights

On the main screen, users can ①quickly switch status, ②receive important notifications, ③view daily pregnancy tips, ④access device connections, and ⑤browse recommended articles. The monitoring page allows users to review historical data, while the profile page provides access to account information, orders, connected devices, and address management.

Guest Mode

When using the app without registration, users can access basic monitoring features. Functions that require personal data will prompt a login or registration message, which triggers ⑩a registration modal.

Jaundice Meter Connection Flow

Users enter the jaundice meter connection flow through entry ①. If Bluetooth is not enabled, the page displays ② a help guide, ③ baby profile management, and ④ a clickable Bluetooth connection guide. Tapping it shows an animation guiding users to enable Bluetooth on their phone. Once Bluetooth is enabled, the device search button becomes active. After tapping it, a search modal appears. When the device is found, the current device information is displayed, and tapping connect will take the user to the measurement screen.

Monitoring Interface

Once successfully connected and on the monitoring screen, users can: ① Use the toggle button to switch the jaundice measurement units. ② View the detection ring, which animates during measurement—green indicates normal levels, orange indicates slightly elevated, and red indicates a dangerous range. ③ Tap the button to view a detailed report or initiate an online consultation. ④ Switch between different data views to provide a more comprehensive record and insights for the user.

When the value reaches a dangerous range, the online consultation button changes to the primary action state.

Fetal Heart Monitor Connection Flow

Similar to the jaundice meter connection flow, users start by tapping ① the fetal heart monitor entry on the main screen. Since the company offers multiple fetal heart monitor models, users need to ③ select the appropriate device—the example here shows the flow for the top model only. After selecting the device, the app searches for it, and once found, users can proceed to connect the device.

If Bluetooth is not enabled, a Bluetooth guide modal will automatically appear. If the device cannot be found, a guide screen is displayed, allowing users to ④ contact official support or ⑤ watch tutorial videos to resolve the issue.

Fetal Heart Monitoring Interface

After connecting the device through ①, users reach ② the onboarding screen for guidance. Tapping the screen or the top-right close button moves to the fetal heart search screen. Following the instructions, ③ starts a 30-second countdown. Once the fetal heart is found, monitoring runs for 20 minutes to obtain results. *If not found, the FHR search flow repeats.

After the monitoring is complete, users are taken to the report screen. ④ allows users to submit the report to a doctor for online review, ⑤ lets them compare reports from different sessions, and ⑥ shows the fetal heart rate and plays the recorded fetal heartbeat.
