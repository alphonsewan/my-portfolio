HM Lab 控制平台

专为 HM Lab 设备设计的可扩展在线控制平台

#Web App

项目概述

HM Studio 专注于自主研发键盘及衍生硬件产品。本项目旨在构建一个 Web-based 硬件控制平台，将品牌官网与设备控制整合，使用户在同一系统中完成从品牌触达到设备连接与参数调节的完整流程。通过统一的视觉语言与交互逻辑，提升不同设备间的一致性，并为未来产品扩展提供支持。

要解决的问题

传统硬件驱动通常需要用户下载安装软件才能使用，流程繁琐。本项目通过 Web-based 的方式，让用户能够实现设备的「即连即用」，无需额外操作即可直接连接和调节设备参数。相比传统驱动，这种方式更便捷、更灵活，也为未来功能扩展和体验创新提供了基础，重新定义了用户与硬件设备之间的交互方式。

最终目的

  • 降低用户的时间成本，实现即连即改的设备驱动体验
  • 将品牌官网与硬件控制整合，增强品牌触达
  • 优化传统驱动模式，让硬件交互更便捷灵活
  • 统一视觉与交互，提升操作连贯性

我的职责

  • 规划整体布局与交互结构，梳理设备操作流程
  • 构建设计系统，包括视觉规范、组件与交互模式
  • 设计官网与设备控制面板的高保真界面
  • 参与动效和小屏 UI 设计，完善用户体验
  • 与开发紧密协作，进行前端视觉优化和小规模代码调试

1.关于优化需求

Discovering Opportunities for Improvement

用户在使用硬件设备时通常需要下载安装驱动并完成复杂配置。本项目通过 Web-based 平台，让用户可以直接连接设备并完成参数调节，实现即连即用的操作体验。下面展示的是我为 Keyboard Driver 和 USB Hub 设计的核心界面与交互流程。

本项目核心目标是构建一个统一的 Web 硬件控制平台。我通过设计系统、交互逻辑和高保真界面，将 Keyboard Driver 与 USB Hub 的操作体验整合在同一平台，并展示在下面的界面案例中。

2.订单管理模块优化

Optimizing the Order Management Experience

在与实际用户的访谈和对系统的观察中，我发现原有的订单管理与设备列表界面存在信息分散、层级混乱、风格不一致等问题。所以我重新梳理了界面结构与交互逻辑，重点优化了关键信息的展示方式与操作效率。 以下为优化前后的界面对；

订单界面

如图中所示（点击可以放大），目前的界面缺少了需求表里面的 ①搜索订单功能模块；然后 ②订单详情里面没办法很好的展示详情数据，只有一些很基础的订单消息从而导致使用者需要再花费额外的时间去查看整体订单的详情；最后就是 ③订单状态目前缺少演示方式，不利于使用者快速发现订单的实际情况。

优化① - 缩进功能

提供界面缩进功能，增加信息的可读空间。

优化② - 详情Modal

订单详细信息不再采取下展方式，改为点击后以弹窗Modal集中显示订单的所有信息和状态以及一些按需求设计的功能

优化③ - 功能添加及颜色调整

①更换成E3A的主题色以及添加公司Logo元素；②增加订单搜索&筛选功能模块； ③订单状态增加，可以更加细致的对应多有可能发生的实际订单情况； ④增加对订单的一些功能

订单结算流程

根据需求，用户可以对入库的设备进行结算操作。其中会显示根据套餐规则和用户实际使用时长进行计算的总额，但是当遇到个别特殊情况时候用户可以有手动调整金额的权限。且可以对当前订单进行备注信息从而可以留下修改原因等信息。

修改金额异常

①②当用户在修改中遇到特殊情况（ex.网络波动）会出现修改不成功的报错并且确认结算按钮会变更为不可用状态

确认结算异常

①当金额修改成功，但是在确认的时候遭遇特殊情况这会弹出报错的Modal。然后用户在点击取消或者800ms之后会回到主界面。 ②当回到主界面的时候，结算异常的订单状态会转为结算异常。

3.设备管理模块优化

Enhancing the Device Management Experience

对于设备模块的优化这里主要展示了录入设备、绑定套餐的演示。新的设计在设备列表界面提供了更加直观性的视觉信息，并且对添加设备&绑定套餐的功能设计了采用了弹窗的modal形式，这能很好的让用户集中处理需要增加和更改的信息

添加单个设备流程（多个设备适用）

图中展示的是新设计的添加设备modal，整个流程同样适用于多设备的同时添加，但是多设备的情况下会多两个不同的功能填选项。添加设备Modal中具有让用户对设备进行类型的选择、名称和图片的定义以及对特定设备进行坐标点定位。在Modal的第二页中，用户可以选填代理商和一些设备设置的功能。

设备绑定套餐流程

此功能是方便用户远程添加设备的套餐绑定情况，可以更加灵活的绑定和更改不同的套餐类型。选中不同绑定的设备时候会弹出符合当前选中设备的对应套餐菜单（目前仅展示为黄疸仪设备），套餐Modal内会具备基础的pagination以及套餐搜索功能。

回顾

E3A订单管理系统优化项目主要是提升系统的用户体验和操作效率。作为ToB项目，重点在于功能优化和流程简化，而非视觉设计。在这个过程中，我根据市场团队的需求优化了现有界面，并设计了新增功能的UI和流程。通过与团队合作，我们解决了界面复杂、操作繁琐的问题。优化后的系统得到了用户好评，操作更流畅，工作效率和满意度显著提升。这次项目不仅提升了我的设计能力，也让我在跨团队合作和快速响应需求方面积累了经验。
